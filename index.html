Agent Skills

A small collection of reusable agent skills for LLM coding harnesses (Claude Code, OpenCode, Cursor, Copilot CLI). Each is a self-contained directory you drop into your skills folder. Install with git clone.

github-pages-porkbun-skill deploydnscert

Deploy a static site to GitHub Pages with a custom domain, where DNS is managed by Porkbun — end to end, including the HTTPS certificate. Bakes in the non-obvious fix for the most common failure: the cert stays null forever because GitHub never starts ACME issuance until you clear and re-add the custom domain via the Pages API.

Includes helper scripts (porkbun.py, pages.py, deploy.py) and deep-dive references on the Porkbun API, the Pages cert lifecycle, and a troubleshooting decision tree.

porkbun-dns dns

Generic DNS record management on Porkbun — create, list, delete, validate. Reads credentials from ~/.secrets/porkbun.env (PORKBUN_API_KEY / PORKBUN_SECRET_KEY). Documents the secretapikey field-name gotcha that breaks most integrations.

Source: skills/porkbun-dns

Installing

Skills are plain directories with a SKILL.md. Clone the repo and point your harness at the skill you want:

# OpenCode / universal
git clone https://github.com/JavaGT/agent-skills ~/.config/opencode/skills/agent-skills

# Claude Code
git clone https://github.com/JavaGT/agent-skills ~/.claude/skills/agent-skills

# Cursor (project-local)
git clone https://github.com/JavaGT/agent-skills .cursor/skills/agent-skills

Each harness discovers skills differently; some read subdirectories directly, others expect a symlink. See your harness's docs.

Credentials

Skills that need secrets read them from ~/.secrets/*.env (dotenv format, chmod 600), never from the repo. Putting them outside the project dir means applications don't auto-load them, and harnesses are more likely to prompt before reading.

~/.secrets/porkbun.env
PORKBUN_API_KEY=pk1_...
PORKBUN_SECRET_KEY=sk1_...
